Lower Bound

You are given a sorted array of integers and a number K.

Lower Bound: Lower bound is a is lowest index where the number is greater than or equal to given target

Example1:

arr = [1,4,11,19,25,36,39,40,42]

target = 18

output: 3

Explanation: The number 19 is greater than 18 and lowest in index

Example2:

arr = [1,4,11,19,25,36,39,40,42]

target = 25

output: 4

Explanation: 25 is equal to target

Explanation

  • we will keep to pointer , low and high
  • low is initialize with 0, low=0
  • high is initialized with n-1, where n is the size of the array, high=n-1, which will be pointing to last element of the array
  • we will calculate mid = (low+high)/2
  • check wether arr[mid]==target
  • Consider three cases based on the comparison between `target` and `arr[mid]`
  • If `target` is smaller than `arr[mid]`, shift the `low` pointer to `mid + 1`. This narrows down our search range to the left half of the array.
  • If `target` is greater than `arr[mid]`, shift the `high` pointer to `mid - 1`. This narrows down our search range to the right half of the array.
  • If `target` equals `arr[mid]`, return `mid` as the lower bound because we've found an element that is less than or equal to `target`.
  • Repeat the steps
